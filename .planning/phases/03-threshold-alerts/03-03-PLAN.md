<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → subscribe route | Untrusted email/term/metric/threshold from the browser |
| email recipient → verify/unsubscribe route | Bearer-capability token in a clicked link |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Spoofing | verify/unsub tokens | mitigate | `crypto.randomUUID()` (122-bit CSPRNG); look up strictly by token; act only on the matched row. |
| T-03-07 | Tampering | alert/verify email HTML | mitigate | `escapeHtml` every interpolated email/term/full_name/reason; `sendEmail` already strips CRLF from subject. Test asserts a hostile term is escaped. |
| T-03-08 | Information Disclosure | subscribe response + verify/unsub responses | mitigate | Generic response regardless of email existence or token validity (no enumeration / no token-probe oracle). |
| T-03-09 | Denial of Service | subscribe endpoint | mitigate | In-memory fixed-window per-IP rate-limit (5/60s) copied from the contact route. |
| T-03-10 | Elevation of Privilege | alerts firing to unconfirmed addresses | mitigate | `verified_at IS NOT NULL` gate (enforced in db.ts queries, asserted by the route test). |
| T-03-11 | Input Validation | metric/kind/threshold/window_days | mitigate | Allow-list normalizers reject anything off-list → 400 before any DB write. |
</threat_model>
