<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/repo-chat | Untrusted fullName, message bodies, and (indirectly) tool arguments |
| README / tool output → Gemini | Untrusted content the model must treat as data, not instructions |
| /api/repo-chat → GitHub / Gemini | Server-side calls using env-var secrets that never reach the client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | client-supplied fullName | mitigate | `isValidFullName` regex + server-side `fetchRepoContext` re-fetch; client repo facts never trusted |
| T-04-05 | Tampering | prompt injection via README / file content / tool output | mitigate | PRD §9 data-not-instructions instruction + `=== UNTRUSTED DATA ===` delimiters in buildSystemPrompt; tool results wrapped in a `data` key with a note |
| T-04-06 | Elevation of Privilege | get_repo_file path traversal / cross-repo read | mitigate | `validateToolArgs` rejects `..` and leading `/`; tool fetches only the current repo; 40KB + binary-extension guard |
| T-04-07 | Denial of Service | rapid chat cost abuse | mitigate | per-IP rate limit 20/60s (no Gemini call on 429) + MAX_TOOL_ROUNDS=3 + 2000-char message cap + last-10-turns cap |
| T-04-08 | Information Disclosure | message bodies / system prompt in logs or storage (INTL-04) | mitigate | no console logging of content, no D1/KV writes; grep acceptance criteria enforce absence |
| T-04-09 | Information Disclosure | GOOGLE_API_KEY / GITHUB_TOKEN | mitigate | keys read from env server-side, never returned to client; absent key → 503 graceful degrade |
| T-04-10 | Tampering | em dash / AI-isms leaking into shipped assistant text | mitigate | `stripEmDashes` server backstop on every streamed chunk + PRD §9 style rules in the system prompt |
</threat_model>
