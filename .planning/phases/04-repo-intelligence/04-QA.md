## Bug found and fixed during QA

**Server components cannot pass event handlers.** `RepoPane.tsx` and the `page.tsx` header (both server components) used `onMouseEnter`/`onMouseLeave` for link hover, which threw "Event handlers cannot be passed to Client Component props" at request render. The build passed because the route is dynamic, so it only surfaced in the browser. Fixed by moving the four hovers to CSS `:hover` classes (`rr-link-back`, `rr-gh-btn`, `rr-tree-link`, `rr-tree-more`) in globals.css, keeping the panes server-rendered. Re-verified: 0 console errors.
