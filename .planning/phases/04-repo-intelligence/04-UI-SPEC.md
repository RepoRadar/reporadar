# Phase 4, UI design contract: Talk to a repo

> Visual and interaction contract for `/chat/[owner]/[repo]` and the "Ask this repo" entry action.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design system

| Property | Value |
|----------|-------|
| Tool | none (manual, extending existing RepoRadar tokens) |
| Preset | not applicable |
| Component library | none |
| Icon library | inline SVG (matching existing RepoCard/RepoRadarApp pattern) |
| Font | Geist Sans (var(--font-geist-sans)) / system-ui fallback |

No shadcn initialization. The project uses hand-rolled components with CSS custom properties. All new components must reuse the existing token set from `app/globals.css`, no new design primitives.

---

## Spacing scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding within chips and badges |
| sm | 8px | Composer inner padding, bubble gaps, tag chip padding |
| md | 16px | Pane inner padding, section gaps within right pane |
| lg | 24px | Outer pane padding on desktop, major section breaks |
| xl | 32px | Desktop gap between left and right panes |
| 2xl | 48px | Vertical breathing room around large content regions |
| 3xl | 64px | Page-level vertical padding |

Exceptions:

- Suggested-prompt chips: 8px vertical, 12px horizontal (compact to fit 4 chips in one row on desktop).
- Composer textarea: 12px top/bottom, 16px left, 48px right (leaves room for the send button overlapping the right edge inside the textarea container).
- Touch targets (send, stop, chip): minimum 44px height, per accessibility requirement.
- The describe-your-build inline input: 12px padding, consistent with the main composer.

## Layout: workspace route `/chat/[owner]/[repo]`

### Desktop layout (>= 768px / `md` breakpoint)

```
┌─────────────────────────────────────────────────────────────────┐
│  [RepoRadar wordmark + mark]    [back to dashboard link]        │  ← page header (48px tall)
├──────────────────────────────────┬──────────────────────────────┤
│                                  │                              │
│   LEFT PANE  ~55% width          │   RIGHT PANE  ~45% width    │
│   (min-width: 320px)             │   (min-width: 280px)        │
│                                  │                              │
│   chat header                    │   repo identity row         │
│   ─────────────────────────────  │   dimension bars (10)       │
│   message list (scrollable)      │   overall score             │
│                                  │   README (scrollable)       │
│   suggested prompt chips         │   file tree list            │
│   (when thread empty)            │                             │
│   ─────────────────────────────  │                             │
│   composer row                   │                             │
│                                  │                             │
└──────────────────────────────────┴──────────────────────────────┘
```

**Exact values:**
- Page max-width: none (full viewport, matching dashboard).
- Left pane: `flex: 0 0 55%` with `min-width: 320px`.
- Right pane: `flex: 0 0 45%` with `min-width: 280px`.
- Gap between panes: 24px (xl token).
- Both panes stretch to `100dvh - page-header-height`. Each pane has its own `overflow-y: auto` scroll context, the panes do NOT scroll together.
- Page header: `height: 48px`, `border-bottom: 1px solid var(--border)`, `background: var(--bg)`, `position: sticky; top: 0; z-index: 10`. Matches dashboard header structure from `RepoRadarApp.tsx`.
- Pane background: `var(--surface)`, `border: 1px solid var(--border)`, `border-radius: 16px` (matches card `rounded-xl`), padding `24px`.
- Right pane: `overflow-y: auto` for the whole pane.
- Left pane: flex column; message list takes `flex: 1; overflow-y: auto`; composer pinned to bottom.

### Mobile / narrow layout (< 768px)

Single-column stack. Two tab buttons ("Chat" and "Repo info") replace the split panes. Chat tab is shown first by default.

- Tab bar: 40px tall, `border-bottom: 1px solid var(--border)`, sits just below the page header.
- Active tab underline: 2px solid `var(--primary)`, label color `var(--fg)`.
- Inactive tab: label color `var(--fg-muted)`.
- Each tab reveals its pane at full width with the same padding (16px on mobile, 24px on desktop).
- No horizontal scroll. Panes stack vertically when tab is active.

## Left pane: conversation

### Chat header (within left pane)

Three lines, top of the pane:

1. **Repo name:** `{repoName}`, `text-xl font-bold`, `color: var(--fg)`. Truncated with ellipsis if too long (`max-width: 100%; overflow: hidden; text-overflow: ellipsis; white-space: nowrap`).
2. **Trust line:** `Grounded in this repo's README, file tree, and RepoRadar scores`, `text-xs`, `color: var(--fg-muted)`. Exact copy, sentence case.
3. **Not-saved note:** `Not saved, conversations end when you close this tab.`, `text-xs font-mono`, `color: var(--fg-dim)`. Exact copy. Rendered with a lock or info icon (SVG inline, 10px, `aria-hidden`).

Header bottom border: `1px solid var(--border)`, `padding-bottom: 12px`, `margin-bottom: 16px`.

### Message list

- Container: `flex-1; overflow-y: auto; display: flex; flex-direction: column; gap: 12px; padding: 4px 0 16px 0`.
- User bubble: right-aligned. Background `var(--surface-3)`, border `1px solid var(--border-strong)`, `border-radius: 12px 12px 2px 12px`, `padding: 10px 14px`, `max-width: 85%`, `text-sm`, `color: var(--fg)`, `align-self: flex-end`.
- Assistant bubble: left-aligned. Background `var(--surface-2)`, border `1px solid var(--border)`, `border-radius: 2px 12px 12px 12px`, `padding: 10px 14px`, `max-width: 100%`, `text-sm`, `color: var(--fg)`, `align-self: flex-start`.
- Assistant markdown rendering: use `react-markdown`. Headings inherit the font. `h1`–`h3` at `text-base font-semibold`, `color: var(--fg)` (not display sizes, answers should scan as body text, not blog posts). `p` at `text-sm leading-relaxed`. `a` at `color: var(--secondary)`, underline on hover, `target="_blank" rel="noopener noreferrer"`. `code` inline: `font-mono text-xs`, background `var(--surface-3)`, `border-radius: 4px`, `padding: 1px 4px`, `color: var(--primary)`. `pre code`: block, `background: var(--surface-3)`, `border: 1px solid var(--border)`, `border-radius: 8px`, `padding: 12px`, `overflow-x: auto`. `ul`/`ol`: `padding-left: 20px`, `gap: 4px`. `strong`: `font-semibold`, same color as parent (no extra emphasis via color).
- Each message row may have a small timestamp in `text-[10px] font-mono color: var(--fg-dim)` on hover, showing "just now / Xm ago", optional, low priority.

### Typing indicator (streaming state)

Three dots animating. Use a CSS `@keyframes` rule that staggers opacity: 1 → 0.3 in a 1.2s loop. Each dot is 6px × 6px, `border-radius: 9999px`, `background: var(--primary)`.

```
◦  ◦  ◦     (pulsing, 1.2s loop, staggered 200ms between each)
```

Wrapped in an assistant bubble with `aria-live="polite"` and `aria-label="RepoRadar is thinking"`. Reduced-motion: replace the animation with a static "Thinking…" text, `color: var(--fg-dim)`, `text-xs font-mono`.

Implementation: add a `.rr-thinking-dot` rule in `globals.css` alongside the existing `rr-blink` / `rr-spinner` rules:

```css
@keyframes rr-think {
  0%, 80%, 100% { opacity: 0.25; transform: scale(0.9); }
  40%           { opacity: 1;    transform: scale(1);   }
}
@media (prefers-reduced-motion: no-preference) {
  .rr-think-dot { animation: rr-think 1.2s ease-in-out infinite; }
  .rr-think-dot:nth-child(2) { animation-delay: 0.2s; }
  .rr-think-dot:nth-child(3) { animation-delay: 0.4s; }
}
```

### Suggested prompt chips

Shown when the message list is empty. Hidden once the first message is sent. A small "Suggestions" label above them is optional, if used: `text-[10px] font-mono uppercase tracking-[0.14em]`, `color: var(--fg-dim)`.

Chip style: matching the existing tag buttons in `RepoCard.tsx`, but slightly larger for touch affordance:
- `border: 1px solid var(--border)`, `background: var(--surface-3)`, `color: var(--fg-muted)`
- `border-radius: 8px`, `padding: 8px 12px`, `text-sm font-mono`
- Hover: `border-color: var(--primary)`, `color: var(--primary)`, `background: rgba(34,197,94,0.08)`
- Focus-visible: `outline: 2px solid var(--primary)`, `outline-offset: 2px`
- Min height: 44px
- All four chips in a `flex flex-wrap gap-2`

**Chip 1:** "Why did you score it this way?"
**Chip 2:** "I'm building something, does this fit?"
**Chip 3:** "Tell me what's so special about this repo."
**Chip 4:** "This sounds like hype. Explain what I'm missing."

**Chip 2 interaction, describe-your-build gate (exact spec):**

Chip 2 does NOT send a message when clicked. Instead, it triggers an inline expansion below the chips:

1. The chip changes appearance: `background: rgba(34,197,94,0.12)`, `border-color: var(--primary)`, `color: var(--primary)`. It renders as a toggle, clicking again collapses the gate.
2. An inline form slides in below the chips (CSS `max-height` transition: 0 → auto, 0.2s ease-out). Contents:
   - A short label: `What are you building?`, `text-xs font-mono`, `color: var(--fg-muted)`, `margin-bottom: 6px`.
   - A textarea (shares the same style as the main composer textarea below, but shorter: 3 rows, `min-height: 72px`). Placeholder: `Describe your project briefly, what it does, how it works, what it needs to connect to.` Autofocus on expand.
   - A submit button: "Check the fit" (primary style, see composer send button below). Keyboard: Enter submits, Shift+Enter adds newline. Disabled when textarea is empty.
3. On submit: the gate collapses, the describe-your-build textarea content is prepended to the chip label in the user bubble sent to the chat: the message sent reads `I'm building something, does this fit? [newline][user's description]`. The assistant then answers.
4. If the user submits an empty description: the textarea shows a `border-color: var(--danger)` pulse, no message sent. No error text needed, the visual flash is enough.
5. `aria-expanded` on the chip button reflects the gate state. The textarea has `aria-label="Describe what you are building"`.

After the first message is sent (from any chip or free typing), chips move to a collapsed "Suggestions" affordance: a small text button "Show suggestions" in `text-xs font-mono color: var(--fg-dim)` at the top of the message list. Clicking re-expands the chip row inline. This is optional for MVP, if time-constrained, chips simply disappear after the first message.

### Composer

Pinned to the bottom of the left pane. Full-width within the pane.

**Structure:**
```
┌────────────────────────────────────────────────┐
│  [textarea, auto-grows to max 6 rows]  [send] │
│  [char count / cap]                            │
└────────────────────────────────────────────────┘
```

- Outer container: `border: 1px solid var(--border)`, `border-radius: 12px`, `background: var(--surface-2)`, `padding: 12px 12px 8px 16px`. Focus-within: `border-color: var(--primary)`, `box-shadow: 0 0 0 2px rgba(34,197,94,0.14)`.
- Textarea: `width: 100%; resize: none; background: transparent; border: none; outline: none; color: var(--fg); font-size: 14px; line-height: 1.5; min-height: 44px; max-height: calc(6 * 1.5 * 14px + 24px)`. Placeholder: `Ask anything about {repoName}…`, uses the short repo name (no owner prefix). Caret color: `var(--primary)`.
- **Keyboard:** Enter sends the message. Shift+Enter inserts a newline. Disabled while streaming.
- Character cap: 2000 characters. Show count when within 200 characters of cap: `text-[10px] font-mono`, `color: var(--fg-dim)`. Over cap: `color: var(--danger)`, send button disabled.
- **Send button:** `inline-flex items-center justify-center gap-1.5 rounded-md border px-3 py-2 text-xs font-medium transition`. Idle: `border-color: var(--primary)`, `background: rgba(34,197,94,0.08)`, `color: var(--primary)`. Hover: `background: rgba(34,197,94,0.20)`, `box-shadow: 0 0 16px var(--primary-glow)`. Disabled (streaming, over cap, empty): `opacity: 0.4`, no hover effect. `aria-label="Send message"`. Icon: a right-pointing arrow (existing SVG arrow pattern or `→` character, 14px).
- **Stop button (streaming only):** Replaces send button while streaming. `border-color: var(--danger)`, `background: rgba(239,68,68,0.08)`, `color: var(--danger)`. Label: "Stop". Hover: `background: rgba(239,68,68,0.20)`. `aria-label="Stop generating"`.

{owner}/{repo}                                      [GitHub repo ↗]

- Stats line: `text-xs font-mono`, `color: var(--fg-muted)`. Language in `color: var(--secondary)`. Separator dots `color: var(--fg-dim)`. Stars formatted identically to `RepoCard.tsx` `formatStars()`.
- Homepage: `text-xs`, `color: var(--secondary)`, underline on hover, `target="_blank"`. Only shown if present.
- **"GitHub repo" link:** Exactly mirroring the `RepoCard.tsx` footer "GitHub repo" action. One instance only. Positioned top-right of the identity row. The identity row "GitHub repo" button is the canonical hub link.

### RepoRadar dimension bars

All 10 dimensions rendered as labeled horizontal bars. Same visual language as the match-score bar in `RepoCard.tsx` and the slider rails in `RepoRadarApp.tsx`.

**Bar anatomy per dimension:**
```
{DIMENSION_META[dim].label}                {score}/100
[═══════════════════░░░░░░░░░░░░░░░░░░░]     (filled bar)
```

- Label: `text-xs font-mono`, `color: var(--fg-muted)`. Acts as a hover trigger for the tooltip (see below).
- Score: `text-xs font-mono font-semibold`, `color: var(--primary)`.
- Rail: `height: 4px`, `width: 100%`, `border-radius: 9999px`, `background: var(--surface-3)`.
- Fill: same `RADAR_GRADIENT` constant as `RepoCard.tsx`, `linear-gradient(90deg, var(--primary), var(--secondary), var(--accent), var(--danger))`. Width = `{score}%`. `box-shadow: 0 0 6px var(--primary-glow)`.
- Container per bar: `display: flex; flex-direction: column; gap: 4px`.
- All 10 bars in a `flex flex-col gap-3` block.
- **Tooltip on label hover:** show `DIMENSION_META[dim].help` text in a `title` attribute on the label element. No custom tooltip component needed, `title` is appropriate here because it is on a small, specific element (not the whole card, which violates the UI contract). Copy is already sentence-case in `DIMENSION_META`.

**Overall score row:**
Below the 10 bars. `border-top: 1px solid var(--border)`, `padding-top: 12px`, `margin-top: 4px`.
```
Overall RepoRadar score                    {overallPct}/100
[full-width bar, same gradient]
```
Label: `text-xs font-mono font-semibold`, `color: var(--fg)`. Score: `color: var(--primary) font-bold`.

### README section

- Section heading: `Readme`, `text-[10px] font-mono uppercase tracking-[0.14em]`, `color: var(--fg-dim)`. Followed by a "View on GitHub" link (`target="_blank"`, `color: var(--secondary)`, `text-xs font-mono`). Heading and link on the same row, space-between.
- README container: `overflow-y: auto; max-height: 480px; border: 1px solid var(--border); border-radius: 8px; padding: 16px; background: var(--surface-2)`. Give this its own scroll context with `tabIndex={0}` and `aria-label="README content, scrollable"` for keyboard navigation.
- README rendered with `react-markdown`. Same markdown styles as assistant messages (see message list above) with one difference: `h1` at `text-lg font-bold`, `h2` at `text-base font-semibold`, `h3` at `text-sm font-semibold`.
- If README is missing: show `No README found for this repo.`, `text-sm font-mono`, `color: var(--fg-dim)`. The chat can still answer; note this in the pane.

### File tree list

- Section heading: `Click around the repo`, same `text-[10px] font-mono uppercase tracking-[0.14em] color: var(--fg-dim)` style.
- Container: `overflow-y: auto; max-height: 240px; border: 1px solid var(--border); border-radius: 8px; padding: 8px 12px; background: var(--surface-2)`. `tabIndex={0}`, `aria-label="Top-level file tree, scrollable"`.
- Each entry: an anchor `<a>` tag. Directories link to `.../tree/HEAD/{path}`, files to `.../blob/HEAD/{path}`. `target="_blank" rel="noopener noreferrer"`.
- Entry style: `flex items-center gap-2 py-1 text-xs font-mono`, `color: var(--fg-muted)`. Hover: `color: var(--secondary)`. Prefix icon: directory gets a folder-style character or inline SVG (real folder icon, must be a recognizable icon, not decorative). File gets a file-page inline SVG. Both at 12px, `aria-hidden`.
- Cap: show top ~30 entries (directories first, then files alphabetically). If truncated, show a note: `{N} more paths, browse on GitHub →` linking to the tree root.

## Entry affordance on repo card

### "Ask this repo" button

Mirrors the existing "Deploy →" button in `RepoCard.tsx`. Added as a second action in the footer row.

**Placement:** Footer row, between the "GitHub repo" link (left) and "Deploy →" (right). The footer row becomes a 3-item flex row: `justify-between` with the GitHub link on the far left, "Ask this repo" in the middle, and "Deploy →" on the far right. On very narrow cards this may wrap, use `flex-wrap: wrap; gap: 8px` and let both buttons share the bottom edge.

**Style:** Same dimensions and shape as "Deploy →":
- `inline-flex items-center justify-center gap-2 rounded-md border px-3 py-2 text-xs font-medium tracking-wide transition`
- Idle: `border-color: var(--secondary)`, `background: rgba(59,130,246,0.08)`, `color: var(--secondary)`.
- Hover: `background: rgba(59,130,246,0.20)`, `box-shadow: 0 0 16px var(--secondary-glow)`.
- Disabled: not applicable (always enabled).

**Icon:** A speech bubble or question-mark-in-bubble SVG, 13px. This must be the real chat/question icon pattern, not decorative. Acceptable: `ChatBubbleLeftIcon` shape (a rectangle with a triangular notch at bottom-left). Inline SVG, `aria-hidden`, 13×13px.

**Label:** `Ask this repo`

**Accessible label:** `aria-label="Ask {owner}/{repo}, opens the chat workspace in a new tab"`. The button text "Ask this repo" is visible; the aria-label adds the repo name for screen readers.

**Behavior:** `onClick` calls `window.open('/chat/{owner}/{repo}', '_blank', 'noopener,noreferrer')`. Stops event propagation (same pattern as the GitHub link). Does NOT call `onDeploy` or `onSelect`.

**Do not** add a `title` attribute to the button or the card element. Scoped tooltip via aria-label only, per the "no card-level title tooltip" contract.

## States

### Loading skeleton (workspace loading)

Both panes show skeleton placeholders while `fetchRepoContext` resolves server-side. Because the page is a server component (page.tsx), the skeleton is the client hydration state before streaming completes. Use React `Suspense` with a `<ChatWorkspaceSkeleton />` fallback.

Left pane skeleton:
- Chat header: two shimmer lines (title-width bar + short bar).
- Message list area: empty (no fake bubbles).
- Suggested chips: four shimmer pill shapes.
- Composer: shimmer rectangle, same height as the real composer.

Right pane skeleton:
- Identity row: two shimmer lines.
- Dimension bars: 10 shimmer bar rows.
- README: shimmer rectangle 320px tall.
- File tree: shimmer rectangle 120px tall.

Shimmer animation: reuse `.rr-shimmer` from `globals.css`:
```css
/* already exists in globals.css */
@keyframes rr-shimmer {
  0% { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}
```
Apply as: `background: linear-gradient(90deg, var(--surface-2) 25%, var(--surface-3) 50%, var(--surface-2) 75%); background-size: 200% 100%; animation: rr-shimmer 1.4s ease-in-out infinite; border-radius: 6px`.

Reduced-motion: remove `animation`, keep the static `background: var(--surface-2)`.

### Repo not found / private / fetch failed

Full-page centered message, no panes rendered. Background `var(--bg)`.

```
This repo is not available.

{owner}/{repo} could not be loaded. It may be private, have been deleted,
or GitHub may be unavailable right now.

[← Back to dashboard]  [Try again]
```

- Title: `text-xl font-bold`, `color: var(--fg)`.
- Body: `text-sm leading-relaxed`, `color: var(--fg-muted)`. Two lines max.
- "Back to dashboard": link to `/`, styled as a secondary button (`border-color: var(--border)`, `color: var(--fg-muted)`).
- "Try again": reloads the page. Primary button style (green border, green text).
- No stack traces, no raw error messages.

### README missing

Right pane renders normally (identity + dimension bars + file tree). README section shows:

```
No README found for this repo.
The chat can still answer from the file tree and RepoRadar scores.
```

`text-sm font-mono color: var(--fg-dim)`. The chat is fully functional.

### Streaming

- Typing indicator (three pulsing dots) renders as an assistant bubble.
- Send button replaced by Stop button.
- Composer textarea: `disabled`, `opacity: 0.6`.
- Suggested chips: hidden (or pointer-events: none).
- Once streaming ends: typing indicator bubble is replaced by the completed message; Stop reverts to Send; textarea re-enables.

### Chat unavailable (no API key)

Right pane renders fully. Left pane renders chat header and message list normally. Composer replaced with:

```
Chat is not available right now.
```

`text-sm font-mono color: var(--fg-dim)`. Shown in a `border: 1px solid var(--border); border-radius: 12px; padding: 16px; background: var(--surface-2)` container in place of the textarea/send row. No send button, no textarea. Suggested chips still show so the user can read them as a guide to what the chat would answer.

### Rate limited

A dismissable banner above the composer (not in the message list):
```
You're sending messages quickly. Give it a few seconds.
```

`text-xs font-mono`, `color: var(--accent)`, `border: 1px solid var(--accent)`, `background: rgba(234,179,8,0.08)`, `border-radius: 8px`, `padding: 8px 12px`. Auto-dismisses after 5s or on next successful send. Dismiss button: `×` at right edge, `aria-label="Dismiss rate limit notice"`.

Do not display HTTP 429 codes or technical error strings to the user.

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Focus order | Page header → tab bar (mobile) → left pane header → message list → chips → composer textarea → send/stop. Right pane: identity → GitHub link → dimension bars → README region → file tree region. |
| Keyboard: send | `Enter` in textarea sends. `Shift+Enter` inserts newline. |
| Keyboard: chip 2 gate | `Enter` on submit textarea submits the description. |
| Keyboard: stop | Stop button is keyboard-reachable. Focus moves to stop when streaming starts. |
| Screen reader: streaming | `aria-live="polite"` on the message list; new assistant messages are announced. |
| Screen reader: typing indicator | `aria-label="RepoRadar is thinking"` on the indicator bubble. |
| Scrollable regions | README container: `tabIndex={0}`, `aria-label="README content, scrollable"`. File tree: `tabIndex={0}`, `aria-label="Top-level file tree, scrollable"`. Message list: `tabIndex={0}`, `aria-label="Conversation"`. |
| GitHub link | `aria-label="Open {owner}/{repo} on GitHub"`, exact pattern from RepoCard.tsx. |
| Entry button | `aria-label="Ask {owner}/{repo}, opens the chat workspace in a new tab"`. |
| Chip 2 toggle | `aria-expanded` reflects gate open/closed state. Gate textarea has `aria-label="Describe what you are building"`. |
| Reduced motion | Typing indicator animation removed. Shimmer animation removed. CSS `@media (prefers-reduced-motion: reduce)` guard on all new keyframe animations. |
| Color contrast | All text uses existing token pairings that meet WCAG AA, `var(--fg)` on `var(--surface)` is #fafafa on #10141b (~14:1). `var(--fg-muted)` on `var(--surface-2)` is ~7:1. Danger/accent on surface meet AA for large text. |
