<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → Cloudflare Insights | pageview beacon sent to a third-party analytics endpoint |
| app → track() backend | event data leaves the client when a backend is configured |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-14 | Information Disclosure (PII leak) | track() event props | mitigate | track() callers pass only safe labels/counts (topic, label, boolean, source-count); `alert_signup` carries NO email; no raw user text. CF Web Analytics is cookieless/no-PII by design (D-11). |
| T-02-15 | Information Disclosure (token/secret) | beacon token in HTML | accept | `NEXT_PUBLIC_CF_BEACON_TOKEN` is a PUBLIC site token by design (CF Web Analytics embeds it client-side); it grants no write/admin access. Documented as owner-supplied. |
| T-02-16 | Tampering (injected script) | beacon script src | mitigate | Hardcoded official CF script URL (`static.cloudflareinsights.com/beacon.min.js`); only the token (from env) is interpolated, JSON-stringified into `data-cf-beacon`. No user-controlled script source. |
| T-02-17 | Denial of Service (blocking user action) | track() in hot paths | mitigate | track() is synchronous-but-trivial (console.debug/no-op) and fire-and-forget; never awaited, never throws (wrap backend send in try/catch when added). |
</threat_model>
